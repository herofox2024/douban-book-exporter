# 豆瓣阅读备份工具

一个 Chrome 扩展，用于抓取并备份豆瓣阅读相关数据。当前支持三类内容：

- 已读书籍
- 长书评
- 读书笔记

抓取结果会保存到本地 `chrome.storage.local`，并支持导出为 `CSV`、`HTML`、`XLSX`。

## 功能概览

- 支持手动输入豆瓣用户 ID
- 支持自动识别当前豆瓣页面用户
- 支持选择抓取目标：已读书籍 / 长书评 / 读书笔记
- 支持自动抓取开关
- 支持本地浏览已保存数据
- 支持按用户隔离存储本地数据
- 支持导出 `CSV`、`HTML`、`XLSX`


### 长书评分页修复

已修复“长书评只能抓到 99 条”的问题。

问题原因：

- 原来的分页逻辑把 `start` 固定按 `50` 递增
- 当接口某一页实际返回条数小于 `50` 时，会把后续一段数据直接跳过去
- 最终导致长书评数量少于接口真实返回值

修复方案：

- 分页停止条件改为以接口返回的 `total` 为准
- 分页偏移改为按“本页实际返回条数”推进
- 长书评继续使用豆瓣 API 抓取，不混入网页列表爬取逻辑

实测：

- 某测试用户的长书评可正常抓取 `162` 条

## 安装方法

1. 克隆或下载本项目到本地
2. 打开 Chrome，进入 `chrome://extensions/`
3. 打开右上角“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择项目中的 `extension` 目录

## 使用方法

### 抓取数据

1. 点击扩展图标打开弹窗
2. 输入豆瓣用户 ID，或直接在豆瓣用户页面中使用自动识别
3. 选择抓取目标
4. 点击“开始抓取”
5. 等待进度完成

### 浏览本地数据

1. 点击“浏览本地数据”
2. 在本地页面中切换已读书籍、长书评、读书笔记
3. 使用搜索框按标题、作者、正文关键词过滤

### 导出数据

1. 在弹窗中选择导出格式
2. 点击“导出数据”
3. 保存导出文件

## 技术说明

### 抓取方式

扩展使用豆瓣移动端 Rexxar API 抓取列表数据：

```txt
GET /rexxar/api/v2/user/{userId}/interests
GET /rexxar/api/v2/user/{userId}/reviews
GET /rexxar/api/v2/user/{userId}/annotations
```

请求不是在 Service Worker 里直接发出，而是通过 `chrome.scripting.executeScript` 注入到豆瓣页面上下文执行。这样可以避免扩展源站跨域请求带来的请求头问题，并自动携带当前登录态 Cookie。

### 长书评分页策略

- 每页请求参数 `count=50`
- 以接口返回的 `total` 作为分页完成依据
- 以本页实际返回条数推进 `start`
- 书评正文通过详情页补抓

### 导出能力

- `CSV`：适合表格处理，带 UTF-8 BOM
- `HTML`：适合直接阅读归档
- `XLSX`：多工作表导出，区分书籍、书评、笔记

## 关键模块

### `DatasetStore`

- 负责本地数据读写
- 按用户 ID 隔离存储
- 保存 `interests / reviews / annotations / updatedAt`

### `DoubanCollector`

- 负责 API 请求
- 负责分页抓取
- 负责正文补抓
- 负责增量合并本地数据

### `Exporter`

- 负责 `CSV` 导出
- 负责 `HTML` 导出
- 负责 `XLSX` 导出

## 常见问题

### 长书评数量为什么可能和网页上看到的不完全一致？

当前长书评以豆瓣 API 实际返回结果为准。插件会尽量完整抓取接口可返回的数据，但不会再混入网页列表爬取逻辑。

### 导出前提示没有数据

请先完成一次抓取，再执行导出。

### 私密数据无法抓取

如果对应数据是私密可见，请先在浏览器中登录对应豆瓣账号。
